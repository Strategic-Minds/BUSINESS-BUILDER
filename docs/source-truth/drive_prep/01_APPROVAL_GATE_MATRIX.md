# Approval Gate Matrix

## Auto-Approved After Build Approval
- Create Drive folders and docs.
- Create GitHub branches and draft PRs.
- Generate source files.
- Generate env placeholders.
- Generate test scripts.
- Run preview-safe API checks.
- Run headless tests when tooling exists.
- Capture screenshots.
- Generate QA reports.
- Auto-fix safe defects.
- Upload build archives and receipts.

## Approval Required
- Merge to main.
- Production deployment.
- DNS or domain changes.
- Live Stripe or Square payment testing.
- Live WhatChimp/WhatsApp send.
- Customer-facing email sends.
- Writing real secrets.
- Destructive database migrations.
- Public SEO content publishing at scale.
- Use of protected XPS assets beyond public linking or authorized assets.

## Morning Handoff Standard
The operator should receive:
- Preview link or local build archive.
- GitHub PR.
- Drive folder.
- QA score and report.
- Screenshot pack.
- Defect/fix log.
- Env readiness report.
- Release gate checklist.
